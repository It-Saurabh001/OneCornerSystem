ARCHITECTURE: MVVM + Clean Architecture (3 Layers)
═══════════════════════════════════════════════════════════════════════════════

📊 PROJECT LAYERS:

1️⃣ DATA LAYER (data/)
   - Models: Data classes jo Firebase se  aate hain
   - Repositories: Firebase operations implement karti hain
   - Remote: API calls (optional)

2️⃣ DOMAIN LAYER (domain/)
   - Repository Implementations: Business logic implement karte hain
   - Use Cases: Specific features ka logic (BaseUseCase extend karte hain)

3️⃣ PRESENTATION LAYER (presentation/)
   - ViewModels: UI state manage karti hain
   - Screens: Jetpack Compose screens
   - States: UI ka state define karti hain
   - Navigation: Screen transitions handle karti hain
